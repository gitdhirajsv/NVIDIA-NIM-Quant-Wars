Executive Research Summary

Research Objective

We asked each model to produce runnable forecasting code, not a market essay. The evaluation then checks whether the generated pipeline can predict responder_6, an anonymized forward-return target, better than a simple mean baseline on the same research sample.

Research Methodology
How We Scored Each Model

Each model's generated notebook was parsed to extract its feature engineering approach. We then ran a standardized XGBoost evaluation on 100,000 rows of real Jane Street production data (partitioned parquet), with an 80/20 train/test split. Target variable: responder_6.

XGBoost hyperparameters were extracted from each model's code (n_estimators, max_depth, learning_rate) via regex, so each model's own tuning choices were respected.

Why responder_6?

The Jane Street competition targets responder_6, a proprietary anonymized financial variable representing an 8-day forward return. Lower MSE = more accurate predictions of future market movements.

The dataset contains 79 anonymized features (feature_00 – feature_78) across multiple date_id batches representing real trading days. Feature engineering — especially the per-date top-quantile flag — is the key differentiator.
